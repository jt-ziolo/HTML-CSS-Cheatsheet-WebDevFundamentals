CSS Box Model, Display and Positioning Tags Cheatsheet

Properties

General

Tag Full Description
opacity Controls opacity. Takes a float value, from 0 (transparent) to 1 (opaque).
visibility Controls whether an element is drawn without changing its influence on layout (with exceptions).

Box Model

Tag Full Description
margin Defines the spacing between this element and bordering elements, starting from the top going clockwise (space separated).
padding Defines the spacing between this element and its own border, starting from the top going clockwise (space separated).
overflow Controls behavior in both directions when an element's content is too big to fit in its block formatting context.
overflow-x Controls behavior in the x direction when an element's content is too big to fit in its block formatting context.
overflow-y Controls behavior in the y direction when an element's content is too big to fit in its block formatting context.
width Sets the width of the element, surrounded by the padding and margin areas.
height Sets the height of the element, surrounded by the padding and margin areas.
min-width Sets the lower limit of the used value of the width property.
min-height Sets the lower limit of the used value of the height property.
max-width Sets the upper limit of the used value of the width property.
max-height Sets the upper limit of the used value of the height property.
box-sizing Sets whether borders and padding are taken into account when sizing the element.

Display and Positioning

Tag Full Description
z-index Sets the z-order of the element and establishes a stacking context for its descendants. Overlapping elements with a larger z-index cover those with a smaller one.
position Sets how an element is positioned.
top
right
bottom
left		 Sets the element's offset relative to the chosen side from either its normal position or its containing block, depending on position.
display An important property that can have complicated effects depending on the options selected. See MDN Web Docs page, here. Basic keywords are covered in the Keywords section below.
float Places an element on the left or right side of the page, allowing text and inline elements to wrap around it.
clear Sets the wrapping behavior of an element relative to neighboring floating elements.

Keywords

General

Tag Full Description
visibility: visible; The element is made visible.
visibility: hidden; The element is made invisible, but is allowed to affect layout as normal.
visibility: collapse; The element is made invisible, but is allowed to affect layout as normal unless it is a table element or collapsed flex item or ruby annotation.

Box Model

Tag Full Description
margin: auto; Allows the browser the freedom to select a suitable margin to use.
overflow: visible; Content is not clipped and will be rendered outside the padding box.
overflow: hidden; Content is clipped where necessary to fit inside the padding box, without automated scrolling.
overflow: clip; Content is clipped where necessary to fit inside the padding box, disallowing any scrolling.
overflow: scroll; Content is clipped where necessary to fit inside the padding box, while providing an option to scroll. The scrollbar is always visible.
overflow: auto; Content is clipped where necessary to fit inside the padding box, while (for most browsers) providing an option to scroll if needed. The scrollbar displaces content.
overflow: overlay; Content is clipped where necessary to fit inside the padding box, while (for most browsers) providing an option to scroll if needed. The scrollbar covers up content.
width: auto;
height: auto;		 The browser is allowed to calculate and select the value for the element. Also works with min- and max-width and height properties.
width: min-content;
height: min-content;		 The intrinsic minimum width. Also works with min- and max-width and height properties.
width: max-content;
height: max-content;		 The intrinsic preferred width. Also works with min- and max-width and height properties.
width: fit-content(...);
height: fit-content(...);		 Sets width to fit the value given, unless it exceeds the max-content value in which case it is set to the max-content value. Also works with min- and max-width and height properties.
box-sizing: content-box; The final width and height of the element is the sum of the set width and height and the thickness of borders and padding.
box-sizing: border-box; The final width and height of the element is the set width and height. The content box shrinks to show borders and padding.

Display and Positioning

Tag Full Description
z-index: auto; Default value. Sets the z-index of this element to 0 without establishing a stacking context for descendants.
position: static; Default value. The element is positioned normally and the top, right, bottom, left and z-index properties are ignored.
position: relative; The element is positioned normally, then offset relative to itself based on the top, right, bottom, left and z-index properties.
position: absolute; The element is removed from normal document flow and offset relative to its closest positioned ancestor based on the top, right, bottom, left and z-index properties.
position: fixed; The element is removed from normal document flow and offset relative to the initial containing block of the viewport, with exceptions if an ancestor has a transform, perspective or filter property set. In printed documents, the element is placed in the same position on every page.
position: sticky; The element is positioned normally, then offset relative to its nearest ancestor with a scrolling mechanism (created by overflow) based on the top, right, bottom, left and z-index properties.
top: auto;
right: auto;
bottom: auto;
left: auto;		 Specifies that the position of the element is based on the value of the opposite side's property (e.g. bottom is used if top is set to auto). If both sides are set to auto, then the element is not moved.
top: inherit;
right: inherit;
bottom: inherit;
left: inherit;		 Specifies that the value is the same as the computed value from the parent element.
display: block; Block-level elements take up the full width of their container, and can have their width and height adjusted.
display: inline; Inline elements take up as little space as possible, flow horizontally, and cannot have their width or height adjusted.
display: inline-block; Inline-block elements can appear next to each other, and can have their width and height adjusted.
float: none; Sets the element to not float. Text and other inline elements will not flow around this element.
float: left; Sets the element to float on the left side. Text and other inline elements will flow around this element.
float: right; Sets the element to float on the right side. Text and other inline elements will flow around this element.
float: inline-start; Sets the element to float on the start side, meaning the left side for left-to-right scripts. Text and other inline elements will flow around this element.
float: inline-end; Sets the element to float on the end side, meaning the right side for left-to-right scripts. Text and other inline elements will flow around this element.
clear: none; Default value. The element will not be moved down to clear past floating elements.
clear: left; The element will be moved down to clear past left-floating elements.
clear: right; The element will be moved down to clear past right-floating elements.
clear: both; The element will be moved down to clear past floating elements on both sides.
clear: inline-start; The element will be moved down to clear past floating elements on the start side, meaning the left side for left-to-right scripts.
clear: inline-end; The element will be moved down to clear past floating elements on the end side, meaning the right side for left-to-right scripts.
